# Stündenglass Brand Assets Portal

A fast, premium **black / white / gold** self-serve portal where retailers and press
download Stündenglass brand & product assets — product photography, lifestyle imagery,
logos, video, packaging and spec sheets — organized by product.

Live at **https://assets.stundenglass.com** (GitHub Pages). It's a **plain static site**
(HTML + CSS + vanilla JS, no build step). Assets are pulled from **Dropbox** by an
hourly sync bot.

---

## How it works

```
Dropbox (Stündenglass folders)
      │   scripts/dropbox-sync.mjs  (GitHub Action, hourly + manual)
      │   • lists each product folder, walks subfolders
      │   • makes a thumbnail for every file (ImageMagick / ffmpeg / poppler)
      │   • mints per-file + per-folder Dropbox share links (real .zip downloads)
      ▼
assets/data/synced.js   ← auto-generated (window.PORTAL_SYNCED). NEVER hand-edit.
      │   overlays onto…
      ▼
assets/data/assets.js   ← hand-authored: brand, product lineup, SKUs, copy, training
      │
      ▼
index.html + assets/css/styles.css + assets/js/app.js   ← the site
      │   Deploy to Pages (.github/workflows/deploy-pages.yml)
      ▼
https://assets.stundenglass.com
```

Downloads ("Download all", "Download folder", "Download selected", per-file) all resolve
to real Dropbox links generated by the sync — nothing large is hosted in this repo.

## Repo layout

| File | What it is |
|------|------------|
| `index.html` | Page shell, meta/OG tags, favicon, cache-bust `?v=` tokens. |
| `assets/css/styles.css` | All styles. Palette is CSS variables at the top (`--ink`, `--gold`, …). |
| `assets/js/app.js` | All behavior (product grid, detail pages, gallery, training/certificate, downloads). One IIFE. |
| `assets/data/assets.js` | **The file you edit** — brand info, product lineup, SKUs, copy, videos, training. |
| `assets/data/synced.js` | **Auto-generated** by the Dropbox sync. Do not hand-edit. |
| `assets/img/` | Logo / brandmark files (currently placeholders — swap for the real brand files). |
| `scripts/dropbox-sync.mjs` | The sync bot (Node, no deps). |
| `.github/workflows/dropbox-sync.yml` | Runs the sync hourly + on manual dispatch. |
| `.github/workflows/deploy-pages.yml` | Deploys the static site to GitHub Pages. |

## Run it locally

```bash
cd stundenglass-asset-portal
python3 -m http.server 8000   # then open http://localhost:8000
```

The Dropbox sync itself runs in GitHub Actions (which installs ImageMagick / ffmpeg /
poppler); you don't need those tools locally to view the site.

## Editing content

Open **`assets/data/assets.js`** — it's heavily commented. Common edits:

- **Product lineup / order** → `PORTAL_CURRENT.stundenglass` and the `PORTAL_PRODUCTS` array.
- **Brand colors/fonts/socials** → `PORTAL_BRANDS.stundenglass`.
- **Contact emails, titles, usage note** → `PORTAL_CONFIG`.
- **SKUs / packaging / MSRP / descriptions / box contents / training** → the labelled
  maps lower in the file (all placeholders right now).

`folders: {}` stays empty on purpose — the Dropbox sync fills each product's folders.

## Wiring a product's Dropbox folder

1. In **`scripts/dropbox-sync.mjs`**, paste the product's Dropbox shared-folder link
   (`https://www.dropbox.com/scl/fo/…`) into the matching `link:""`. `name` must match the
   product `name` in `assets.js` exactly. Entries with an empty link are skipped, so the
   sync always runs cleanly.
2. Commit & push. Run the **Dropbox sync** workflow (Actions tab → Run workflow), or wait
   for the hourly run. It regenerates `synced.js` and the deploy publishes it.

### Dropbox credentials (GitHub Actions secrets)

The sync authenticates with the **same Dropbox app as the G Pen portal**. Add these three
repo secrets (Settings → Secrets and variables → Actions) — never commit them:

- `DROPBOX_APP_KEY`
- `DROPBOX_APP_SECRET`
- `DROPBOX_REFRESH_TOKEN`

Scopes required (already granted on the app): `files.metadata.read`, `files.content.read`,
`sharing.write`.

## Deploy / DNS

GitHub Pages (Actions-based) publishes on every push to `main` and after each Dropbox
sync. Custom domain is set via the `CNAME` file (`assets.stundenglass.com`) plus a DNS
record at the registrar:

```
CNAME   assets   →   <repo-owner>.github.io
```

Enable HTTPS in Settings → Pages once the certificate is issued.
